Design — Modelo de dados e merge

Como os dados das duas fontes se combinam

Interface FuncionarioRelogio

interface IFuncionarioRelogio {
matricula: string;
nome: string;
empresa: string;
local: string;
cargo: string;
ativo: number; // 0 | 1
relogiosCadastrado: number | null; // null = ainda sem dados
relogiosAtivo: number | null;
fonte: 'local' | 'api' | 'ambos';
}

Estratégia de merge (chave: matricula)

Local + API
Usa dados do banco local (mais completos: empresa, local, cargo, ativo). Marca fonte: 'ambos'. Ignora campos redundantes da API.
Só local
Funcionário existe no banco mas não está na API (ex: cadastrado manualmente). Marca fonte: 'local'.
Só API
Funcionário existe na Stefanini mas não no banco local. Mapeia os campos disponíveis (CodigoFuncionario → matricula, NomeFuncionario → nome). Empresa/local/cargo ficam vazios. Marca fonte: 'api'.

Campo fonte na tabela

O campo fonte fica no modelo mas não aparece como coluna na tabela. Pode ser usado futuramente para destacar linhas de origem desconhecida ou para debug.
